CardShark — Card Catalog Format

The CardShark iOS app fetches this catalog to power the "Add from Catalog" picker. Manually maintained. → card-catalog.json

Not financial advice. Reward rates change often. These entries are a curated starting point — always verify a card's current terms with the issuer. The app shows a "verify current terms" note when a card is added.

File shape

{
  "version": 1,               // schema version (integer)
  "generatedAt": "2026-06-22T00:00:00Z",  // optional ISO-8601 timestamp
  "cards": [ Card, Card, ... ]
}

Unknown fields are ignored, so the file is forward-compatible. The app re-validates every entry and falls back to its bundled copy if the file is missing, malformed, or unreachable.

Card

FieldTypeNotes
displayName *stringe.g. "Amex Gold". Cards with an empty name are dropped.
shortNamestring?Compact label for pills; falls back to displayName.
issuerstringFree text, e.g. "American Express", "Chase", "Citi". Used to group the picker.
networkenumvisa · mc · amex · discover
currencyenumcashback · chaseUR · amexMR · capitalOneMiles · citiTYP · other
baseRate *numberEverywhere earn rate (see rate semantics).
isPointsBase *booltrue → baseRate is points/$; false → cashback fraction.
hasAnnualFeebool?Ranking tiebreaker (prefer no-fee on ties).
tintColorHexstring?#RRGGBB. Invalid values are replaced with a default color.
rules[Rule]Category bonus rules (may be empty).

Rule

FieldTypeNotes
category *enumdining · grocery · gas · travel · drugstore · transit · streaming · online · other
rate *numberSee rate semantics.
isPoints *booltrue → points/$; false → cashback fraction.
requiresActivationbool?Rotating/quarterly category needing enrollment.
isActiveThisQuarterbool?User toggle (leave false in the catalog).
hasCapbool?Whether the bonus has a spend cap.
capExhaustedbool?User toggle (leave false in the catalog).
capNotestring?Free text, e.g. "$1,500/qtr".
isExcludedbool?Bonus excluded at some merchants (rare).

Rate semantics

When isPoints/isPointsBase is true, rate is points per dollar (4.0 = 4×). When false, it is a cashback fraction (0.02 = 2%). Validation bounds: points rates 0…100, cashback fractions 0…1 — entries outside these are dropped.

Example card

{
  "displayName": "Amex Gold",
  "issuer": "American Express",
  "network": "amex",
  "currency": "amexMR",
  "baseRate": 1.0,          // 1x points everywhere
  "isPointsBase": true,
  "hasAnnualFee": true,
  "tintColorHex": "#FF9F0A",
  "rules": [
    { "category": "dining",  "rate": 4.0, "isPoints": true, "hasCap": true, "capNote": "4x on first $50k/yr dining" },
    { "category": "grocery", "rate": 4.0, "isPoints": true, "hasCap": true, "capNote": "4x on first $25k/yr U.S. supermarkets" }
  ]
}

* required · fields marked ? are optional.
